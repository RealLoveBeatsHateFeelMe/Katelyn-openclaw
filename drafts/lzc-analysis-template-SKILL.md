# 八字交叉验证分析 — 完整方法论 + Taylor Swift 标杆

## 核心原则（3 条铁律）

1. **引擎数据 ≠ 分析。交叉验证 = 分析。** 引擎输出只是"假设"，真实事件才是"证据"。没有证据的假设不输出。
2. **每个引擎关键词必须对应至少 1 个具体真实事件**（年份+人名+事件名）。"个人魅力强"这种话等于没说 — 必须列出 Big Machine 版权战、Eras Tour 146场这种级别的事实。
3. **先 research 再分析。** 拿到 API 数据后第一件事不是写分析，是去搜这个人的 Wikipedia / 新闻 / 感情史。没有 research 的分析是垃圾。

---

## 分析 5 步流程

Step 1: 调 /v1/agent/profile API 拿八字数据

Step 2: Research 真实生平（你目前跳过的步骤！）
- Wikipedia 传记全文
- 近 5 年重大新闻（每年至少 1 件）
- 感情史完整时间线（历任对象 + 开始/结束年份 + 属相）
- 职业里程碑（年份+成就，按十年分段）

Step 3: 交叉验证（核心价值所在）
- 每个维度：引擎输出 vs 真实事件 → 判定（准/不准/待验证）
- 找"最震撼的验证点"（时间精确到半年 / 描述前所未有的事件）

Step 4: 评级
- 每维度 1-5 星
- 综合评级
- 列出"产品 demo 最强验证点" TOP 3-5（**只要跟大运有关的验证点**，大运是核心卖点）

Step 5: 输出分析文章（按下方模板）

## 每个维度的验证方法论

### 读 analysis_guide

API 返回的第一个字段就是 analysis_guide，它告诉你阅读顺序：
- Step 1：先看 liuqin（六亲助力）+ dayun（大运走势 + turning_points 转折年）— 这是命盘骨架：一辈子的底色
- Step 2：再看 feeling（感情窗口）+ liunian（流年评分）— 这是命盘血肉：具体年份的天气
- 核心概念：大运 = 环境（10年底色），流年 = 天气（某年好坏）。先理解环境再看天气。

### 一、性格（personality.dominant_traits）

**错误做法**：
"正偏印复杂 86.7%: 个人魅力和领导力强" ← 笼统废话

**正确做法**：
1. 拿到 dominant_traits 列表（如 七杀 46.7%）
2. 理解每个十神代表什么性格：
   - 七杀 = 执行力、决断力、抗压、不服输
   - 食神 = 温和、享受生活、产出、表达
   - 伤官 = 创新、叛逆、表达欲强
   - 正印 = 贵人缘、学习能力、内敛
   - 偏印 = 偏门思维、灵感、独处
   - 正财 = 稳健、努力型
   - 偏财 = 机会嗅觉、冒险
   - 比劫 = 独立、竞争意识
   - 官杀 = 纪律、目标感、承受压力
3. 对每个特质列 3-5 个具体真实事件（年份+事件名+为什么对上这个十神）
4. 日主意象（天干）找一个贴切的比喻：
   - 甲木=大树、乙木=藤蔓、丙火=太阳、丁火=蜡烛
   - 戊土=大山、己土=田园、庚金=宝剑、辛金=珠宝
   - 壬水=大海、癸水=雨露

### 二、六亲助力（liuqin）

**错误做法**：
"无数据" ← 即使空也有含义

**正确做法**：
1. 每个六亲翻译成人话（API 的 source 字段已经给了）：
   - 印 = 母亲/长辈/贵人/老师
   - 比劫 = 兄弟姐妹/同辈朋友/同学同事
   - 食伤 = 才艺/技术产出/子女
   - 财(男) = 妻子/伴侣 + 钱
   - 官杀(女) = 丈夫/男友 + 领导
2. 有数据的：strength + source → 对照真实的人（谁是贵人？具体帮了什么？）
3. **只列有数据的六亲组就行**，不用强制列全。但如果某个重要的六亲（比如印/贵人）是空的，且这个"空"本身很有洞察价值（比如说明此人白手起家），就提一句。
4. 总结"关键洞察"：这个人的成功路径是靠什么驱动的

**标杆（Taylor Swift）**：
- 比劫（同辈朋友）：用神有力 → Swifties = 史上最团结粉丝群；Squad 闺蜜文化
- 食伤（才艺产出）：用神有力 → 14张专辑、400+首歌、Grammy x4
- 印（长辈贵人）：原局没有印星 → 妈妈2015确诊癌症（有心无力的字面含义）
- 关键洞察：Taylor 的成功路径是同辈+才艺驱动，非贵人提携

### 三、大运（dayun）— 最关键的维度，分发文章的核心卖点

**为什么大运最重要**：所有非常成功的人，标配是连续 30-50 年好运。这个信息是产品最大的视觉冲击。分析文章的核心叙事线就是：
- 这个人从什么时候开始起运？
- 能延续到什么时候？
- 每步大运分别代表什么特点？

**必须输出的 3 个核心信息（缺任何一个就不合格）**：

1. **好运总时长**：从 dayun.turning_points 算出连续好运跨度。看哪几步 is_good=true 连在一起。例如 Taylor Swift: "1997-2066，70年好运" — **这是文章标题级别的信息，必须在大运段开头大字写出来**。如果好运不连续，标出哪段好哪段差。
2. **起运年份 + 那年发生了什么**：第一步好运大运的 start_year，映射到几岁、那年的真实事件。这能回答"这个人什么时候开始起飞的"。
3. **每步大运的十年特点**：API 返回的 dayun 里每一步都要展开。**不能合并大运段！** API 返回几步就写几步，每步 10 年单独分析。每步必须有至少 3 个真实事件。

**错误做法**：
5步大运4步是"-" ← 等于没分析

**正确做法**：

1. 第一件事：算好运总时长。数 dayun 里 is_good=true 的连续步数 x 10年。
2. 起运那一年发生了什么：research 那个年份这个人的真实事件。
3. 每步大运逐段分析（格式）：

#### 大运N: [干支] [YYYY-YYYY]（X-X岁）— [好运/一般]：[十年关键词]

| 引擎关键词 | 真实事件 |
|-----------|---------|
| 关键词1 | 具体事件（年份+人名+成就） |
| 关键词2 | 具体事件 |
| 关键词3 | 具体事件 |

**判定：[准确/一般/不准]。** [一句话总结为什么对上/没对上]

4. 每步大运至少列 3 个真实事件。没有事件 = 你没 research 够。
5. turning_points 是最值钱的验证点：大运好坏切换的年份 vs 这个人生命中的重大转折。

### 四、流年（liunian）

**错误做法**：10 年只填 2 个事件

**正确做法**：
1. 过去 5 年：每年必须查这个人那年发生了什么（搜 "[名人] [年份]" 的新闻）
2. 当前年：重点分析（最新热点）
3. 未来 4 年：标记"待验证"
4. 引擎 rating 与真实事件对照：
   - "凶" → 那年有没有官司/伤病/丑闻/重大损失？
   - "好" → 那年有没有冠军/升职/突破/重大成就？
   - "波折" → 那年有没有起伏？先好后坏？
   - "前X后Y"（如"前一般后好"）→ 上半年和下半年能不能区分开？这是最强验证点！

**标杆（Taylor Swift 2022: "前一般后好"）**：
上半年相对平静 → 10月 Midnights 发行，打破所有首周销量记录 → "前一般后好"精确对应了 10月发行的时间节奏。

---

### 五、感情（feeling）

**错误做法**：只列引擎信号，零真实对照

**正确做法**：
1. 先 research 这个人的完整感情时间线（历任对象+开始/结束年份+属相）
2. 对每个有信号的年份：引擎信号 vs 真实事件
   - "遇到合适的人" → 那年真的开始了新关系吗？
   - "感情变动" → 那年真的分手/离婚了吗？
   - "有暧昧但难成正果" → 那年有短暂关系但没结果吗？
3. 婚配验证（如果知道对象生日）：
   - API 返回 bazi.yongshen_elements → 对应的属相就是推荐范围
   - 历任对象的属相在推荐范围内 → 验证"匹配"
   - 不在推荐范围内 → 验证"不匹配"（结果通常是短暂/痛苦）
   - 正向+反向都要做（推荐的长久 + 不推荐的短暂 = 双向验证）

## 反模式清单（7 个必须避免的错误）

1. 复述引擎数据当分析 — 引擎数据只是假设，research 真实事件才能验证
2. "无数据"就跳过 — 空也有含义（"没有贵人提携"本身就是洞察）
3. 笼统描述（"个人魅力强"） — 必须列具体事件（年份+人名+事件名）
4. 大运全写"-" — 每步大运必须 research 那十年的重大事件，至少 3 个
5. 流年只填 2 个事件 — 过去 5 年每年必须查
6. 感情窗口只列信号 — 必须查真实感情史对照
7. 没有评级和"最强验证点" — 综合评定 + TOP 3 验证点是文章的灵魂
8. **合并大运段** — API 返回几步大运就写几步，不能把 20 年合成一段。每步 10 年必须单独分析
9. **没算好运总时长** — "连续 X 年好运"是标题级信息，缺了整篇文章没有核心卖点

---

## 质量自检（输出前必须过）

交出分析前问自己 5 个问题：

1. 大运好运总时长算了吗？起运年份标了吗？
2. 每步大运至少 3 个真实事件吗？
3. 过去 5 年流年每年都有真实事件对照吗？

5. "产品 demo 最强验证点" TOP 3 列了吗？

**任何一个"否" → 不要交出，继续 research。**
