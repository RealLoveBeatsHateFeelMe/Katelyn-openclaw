# Celebrity Playbook — Selection & Verification Framework

## Purpose

Define the standard for selecting and verifying celebrities for Likely You engine demos. Every analysis must meet these criteria or be rejected.

## Selection Criteria

### Must-Have (All Required)

| Criterion | Details |
|-----------|---------|
| **Public birthday** | Full YYYY-MM-DD from Wikipedia or official bio. No estimated/disputed dates. |
| **Verifiable career events** | 10+ years of documented milestones with specific dates/years |
| **Active audience** | Currently relevant — trending, in the news, or major fanbase |
| **Career arc variety** | Clear peaks and valleys, not a flat trajectory |
| **Multiple data sources** | Events confirmable from 2+ independent sources |

### Nice-to-Have

- Birth time available (enables more precise engine analysis)
- Major life transitions visible (career pivots, comebacks, controversies)
- Cross-domain recognition (known beyond their niche)

## The Taylor Swift Benchmark

Taylor Swift is the gold standard for engine verification:

- **70-year good fortune pattern** — engine maps sustained success across decades
- **6-dimension verification** — all 6 engine dimensions align with real events
- **Public timeline density** — hundreds of verifiable career events with exact dates
- **Cultural penetration** — audience immediately understands the references

Any new celebrity analysis should aim for Taylor Swift-level verification density. If you cannot find at least 3 real events per engine dimension, the celebrity is not suitable.

## ENGINE vs REALITY Framework

This is the core methodology. For every engine output:

```
ENGINE says: [exact engine output for this dimension]
REALITY shows: [specific real event with date and source]
VERDICT: Match / Partial / Mismatch
```

### Rules

1. **ENGINE first** — always present what the engine predicted before showing reality
2. **Specificity required** — "had a successful career" is not evidence. "Won Grammy for Album of the Year on Feb 5, 2016" is evidence.
3. **Date everything** — every real event needs a year at minimum, exact date preferred
4. **Source everything** — Wikipedia, official press, news articles
5. **Never cherry-pick** — if ENGINE says something that does NOT match reality, report it honestly

## Top Verification Points Ranking

After completing a full analysis, rank the strongest verification points:

```
Top Verification Points

1. [Dimension]: ENGINE said [X], REALITY confirmed [Y] — [why this is compelling]
2. [Dimension]: ENGINE said [X], REALITY confirmed [Y] — [why this is compelling]
3. [Dimension]: ENGINE said [X], REALITY confirmed [Y] — [why this is compelling]
```

These become the "highlight reel" for product demos. Prioritize:
- Specific date matches (engine window aligns with real event year)
- Counter-intuitive predictions that turned out true
- High-stakes life events (marriage, major career pivot, health crisis)

## Quality Gate

Before publishing any analysis:

- [ ] All 6 dimensions addressed
- [ ] Minimum 3 real events per dimension
- [ ] ENGINE presented before REALITY (no retrofit)
- [ ] All events dated and sourced
- [ ] Mismatches honestly reported
- [ ] Top verification points ranked
- [ ] Brand voice compliance checked (see lzc-brand-voice)
